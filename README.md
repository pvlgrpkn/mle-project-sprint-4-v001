# Подготовка виртуальной машины

## Склонируйте репозиторий

Склонируйте репозиторий проекта:

```
git clone https://github.com/yandex-praktikum/mle-project-sprint-4-v001.git
```

## Активируйте виртуальное окружение

Используйте то же самое виртуальное окружение, что и созданное для работы с уроками. Если его не существует, то его следует создать.

Создать новое виртуальное окружение можно командой:

```
python3 -m venv env_recsys_start
```

После его инициализации следующей командой

```
. env_recsys_start/bin/activate
```

установите в него необходимые Python-пакеты следующей командой

```
pip install -r requirements.txt
```

### Скачайте файлы с данными

Для начала работы понадобится три файла с данными:
- [tracks.parquet](https://storage.yandexcloud.net/mle-data/ym/tracks.parquet)
- [catalog_names.parquet](https://storage.yandexcloud.net/mle-data/ym/catalog_names.parquet)
- [interactions.parquet](https://storage.yandexcloud.net/mle-data/ym/interactions.parquet)
 
Скачайте их в директорию локального репозитория. Для удобства вы можете воспользоваться командой wget:

```
wget https://storage.yandexcloud.net/mle-data/ym/tracks.parquet

wget https://storage.yandexcloud.net/mle-data/ym/catalog_names.parquet

wget https://storage.yandexcloud.net/mle-data/ym/interactions.parquet
```

# Расчёт рекомендаций

Код для выполнения первой части проекта находится в файле `recommendations.ipynb`. Изначально, это шаблон. Используйте его для выполнения первой части проекта.

# Сервис рекомендаций

Код сервиса рекомендаций находится в файле `recommendations_service.py`.

Перед запуском сервиса нужно получить локальные артефакты первой части проекта. Сервис использует файлы:
- `items.parquet`
- `top_popular.parquet`
- `personal_als.parquet`
- `similar.parquet`
- `recommendations.parquet`
- `metrics.json`

Если артефакты ещё не созданы, их можно собрать командой:

```
python3 run_part1.py
```

После этого сервис можно запустить из корня репозитория командой:

```
uvicorn recommendations_service:app --host 0.0.0.0 --port 8000
```

По умолчанию сервис ищет parquet/json-артефакты в текущей директории репозитория. При необходимости можно указать другой путь через переменную окружения `RECSYS_DATA_DIR`.

Сервис предоставляет три endpoint:
- `GET /health` — проверка готовности и загрузки артефактов;
- `POST /events` — запись онлайн-события прослушивания в память сервиса;
- `GET /recommendations?user_id=<id>&k=<n>` — выдача рекомендаций для пользователя.

Пример записи онлайн-события:

```
curl -X POST http://127.0.0.1:8000/events \
  -H "Content-Type: application/json" \
  -d '{"user_id": 3, "track_id": 26}'
```

Пример запроса рекомендаций:

```
curl "http://127.0.0.1:8000/recommendations?user_id=3&k=5"
```

### Стратегия смешивания онлайн- и офлайн-рекомендаций

Сервис использует простую и детерминированную схему смешивания:
- если у пользователя есть онлайн-история внутри текущего процесса сервиса, берутся последние прослушанные треки;
- для них подбираются похожие треки из `similar.parquet`;
- онлайн-кандидаты очищаются от дублей и уже прослушанных пользователем треков;
- затем они домешиваются к офлайн-рекомендациям из `recommendations.parquet`;
- если для пользователя нет ранжированных офлайн-рекомендаций, используется `personal_als.parquet`;
- если и персональных рекомендаций нет, сервис переходит на fallback через `top_popular.parquet`.

В текущей реализации сервис добавляет до двух онлайн-кандидатов в верхнюю часть выдачи и заполняет оставшиеся позиции офлайн-рекомендациями. Такой подход позволяет учитывать свежую онлайн-активность пользователя, не отказываясь от уже рассчитанной офлайн-персонализации.

# Инструкции для тестирования сервиса

Код для тестирования сервиса находится в файле `test_service.py`.

Для автоматического тестирования сервиса выполните команду:

```
python3 test_service.py
```

Скрипт сам:
- поднимает `uvicorn` в отдельном процессе;
- ждёт готовности endpoint `GET /health`;
- проверяет три сценария:
  - пользователя без персональных рекомендаций;
  - пользователя с персональными рекомендациями и без онлайн-истории;
  - пользователя с персональными рекомендациями и с онлайн-историей;
- завершает сервис после теста.

Результаты каждого запуска сохраняются в файл `test_service.log`. Если все проверки прошли успешно, в конце лога будет строка `All service scenarios passed successfully.`.
